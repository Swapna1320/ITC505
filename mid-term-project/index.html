Escape from Cyber Academy

Game Addendum

1. Process Overview

I started with a formula of the story's basic framework that included a starting point along with no fewer than eight distinctive finishing points that met the requirements. I created a fantastical storyline about escaping from the Cyber Academy which properly matches the technological theme while it opens up opportunities for multiple branching outcomes.

2. Planning & Structure

I organized the story into a system that uses decision trees as its structure. For each decision made, the system leads to different scenes which contain their own textual content and visual display together with user selection options. I designed the JavaScript by using a main story object to maintain logical structure and data functionality.

3. Creative Choices

  • Setting: Students at the tech academy can learn various possibilities such as terminal hacking and vent exploration as well as data theft.
  • Design: The user interface presented both straightforward design and crystal-clear elements through presenting one scene image and easily identifiable buttons.
  • Endings: Various outcomes were included such as escape from the forest and being caught by police or entering the academy and remaining in hiding forever.

4. Technical Highlights

  • JavaScript DOM manipulation: Enabled dynamic change of the story text alongside image display and choice modification.
  • Reusable Functions: The application contains three reusable functionsâ€”startGame combined with updatePage and endGame.
  • Dynamic Image Display: I made sure the images display new content throughout the story to enhance the immersive experience.
  • Timestamp: The application displays the timestamp of the most recent update through document.lastModified.

5. Challenges

A key challenge emerged from the relationship between code selections and their corresponding outcomes since code organization required attention. A single-story object simplified both maintaining narrative logic and enabling growth of the story content. The main difficulty for this project involved making sure each possible conclusion generated its unique visual display as well as verifying the accessibility of all narrative routes.
