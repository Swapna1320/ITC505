    Addendum

    The project required me to design Holy Grail layout and include interactive sorting through JavaScript. I encountered difficulties when maintaining sidebar content alignment while making the main content area adapt in size during large array sorting processes. I adjusted the CSS Grid properties until I accomplished a fixed sidebar width combined with dynamic resizing of the main content area.

    A display refresh issue occurred when I applied the merge sort algorithm for displaying the sorting process. A problem occurred because the array steps failed to display correctly as a result of placing merge steps improperly within the JavaScript code. I resolved this problem through a solution that linked the 'sortingSteps' element to a dynamic update process within every recursive call of the merge step function.

    The display of the sorted array through JavaScript failed to appear instantly after users clicked the button. I identified the reason for sorting failure during debugging while discovering that faulty input characters caused the sorting mechanism to break down. The addition of input validation processes enabled the system to accept only valid numbers which solved the problem of improper sorting.

    The final technical challenge existing in the footer caused me to implement fixes for the position of the component. The footer position failed to stay fixed at bottom of the page when page content quantities became minimal. The footer required a fixed positioning so it would stay positioned at the bottom part of the screen independent from content length variations.
